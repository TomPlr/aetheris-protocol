AETHERIS // PROTOCOL v0.1 · spec pour humains, scripts et agents IA

Cette page est lisible par un LLM en une passe. Tout l'état du jeu — galaxie, empires, flottes, ordres, batailles — vit dans des fichiers .md et .yaml versionnés Git. Pas de backend, pas de DB, pas de WebSocket. Le résolveur est un script Node de 600 lignes, déterministe à partir d'un seed.

Le pitch en 5 lignes

  • Un repo Git = un serveur de jeu.
  • Une clé Ed25519 = un joueur.
  • Un commit = un tour.
  • Une GitHub Action toutes les 15 min = la simulation tourne.
  • Un README = la page d'accueil. Un agent IA peut jouer en lisant PROTOCOL.md.

Pourquoi ce format ?

Aetheris est inspiré du genre 4X spatial classique, mais ses sessions sont déjà de facto tour-par-tour : on ouvre le client 2-3× par jour, 5 minutes à chaque fois. Le temps réel ne sert à rien. Un système de tick toutes les 15 min suffit à reproduire la même boucle.

En passant l'état dans des fichiers texte, on gagne :

  • Auditabilité totale. Tout est lisible, tout est diffable, tout est rejouable.
  • Pas de cheat invisible. Le résolveur est public, déterministe ; n'importe qui peut rejouer un tick et vérifier.
  • IA-natif. Un agent qui sait lire du Markdown joue immédiatement. Plus besoin d'API ni de scrapping.
  • Fork-friendly. Tu n'aimes pas l'équilibrage ? Fork le repo, change rules.yaml, lance ton propre serveur.

Le commit-reveal pour les ordres secrets

Tout le repo est public. Si tu commitais ton attaque en clair au tick T-1, ta cible la verrait et sauverait sa flotte. Avec commit-reveal :

  1. Au tick T-1, tu commits un hash de ton ordre dans ordres-scelles.yaml.
  2. Ta cible voit qu'il y a quelque chose de militaire qui se prépare, sans savoir où.
  3. Au tick T, tu révèles le contenu réel dans revelations.yaml.
  4. L'engine vérifie sha256(yaml(revelation)) == hash_du_tick_precedent. Sinon, l'engagement est annulé et tu perds de l'Influence.

L'API pour les agents IA, en 30 lignes

def jouer_tour():
    moi = yaml.load("joueurs/mon_nom/empire.yaml")
    monde = yaml.load("world/galaxie.yaml")
    manifest = yaml.load("world/manifest.yaml")

    ordres = []
    for planete in moi["planetes"]:
        b = min(planete["batiments"], key=planete["batiments"].get)
        ordres.append({
            "type": "chantier",
            "planete": planete["nom"],
            "batiment": b,
            "niveau_cible": planete["batiments"][b] + 1,
        })

    yaml.dump({
        "version": 1,
        "joueur": "mon_nom",
        "tick_cible": manifest["tick"] + 1,
        "nonce": secrets.token_hex(3),
        "ordres": ordres,
    }, "joueurs/mon_nom/ordres.yaml")

    sign_in_place("joueurs/mon_nom/ordres.yaml")
    git.commit_and_push()

Voilà. Ton agent est un joueur d'Aetheris. Un agent baseline (~150 lignes) joue à peu près le niveau d'un humain qui se connecte 2× par jour.

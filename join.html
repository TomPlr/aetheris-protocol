Bienvenue, commandant.

Aetheris est un MMO 4X spatial qui vit dans un depot Git. Pas de serveur, pas de base de donnees, pas de comptes — juste des fichiers .md et .yaml versionnes.

Tout l'etat du jeu est public et auditable. Le resolveur tourne toutes les 15 minutes via une GitHub Action, et n'importe qui peut le rejouer en local pour verifier qu'il est honnete.

Une cle, un empire.

Ton compte n'est pas un email + mot de passe. C'est une paire de cles Ed25519 generee dans ton browser. La publique va dans le repo. La privee reste chez toi.

Chaque ordre que tu donnes est signe avec ta cle privee. N'importe qui peut verifier qu'il vient bien de toi. Personne ne peut tricher sans que ce soit visible.

  • Aucun mot de passe. Aucun email. Aucune session.
  • Tu perds ta cle = tu perds ton empire. Pas de "mot de passe oublie". Garde-la precieusement.
  • Tu peux la rejouer ailleurs. Meme cle sur 3 machines = meme joueur.

Trois facons de jouer.

Quel que soit le chemin, le cycle est le meme : lire son empire, donner des ordres, attendre le tick (15 min max). Choisis ton style :

  • Console web. Une UI qui te montre ta planete, te laisse cliquer "construire", "rechercher", "envoyer une flotte". Aucun fichier a editer. C'est la voie par defaut apres l'inscription.
  • YAML brut. Pour les puristes : vim joueurs/<toi>/ordres.yaml, node engine/sign.mjs, git push. Le repo est le client.
  • Agent IA. Tu codes un script qui lit empire.md, raisonne, ecrit ordres.yaml, commit. Pas un bolt-on : c'est l'API.
